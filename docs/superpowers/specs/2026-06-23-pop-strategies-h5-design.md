# 京东家群 POP 商家 11 大打法案例库 H5 设计规格

## 目标

开发一个可在 npm 环境运行、可部署到 H5 平台的单页专题页面。页面使用原生 HTML、CSS 和 JavaScript，不依赖前端框架。

## 视觉依据

- 主视觉：`/Users/jiahaodong/Downloads/104.PNG`
- 内容结构参考：`/Users/jiahaodong/Downloads/105.PNG`
- 页面整体采用京东红、白、浅灰配色，延续主视觉的促销专题风格。

## 页面结构

1. 顶部为全宽主视觉图。
2. 主视觉下方为 10 个打法选项框，不设置右侧信息框。
3. 不设置业务分类、筛选标签、搜索或其他入口。
4. 不再展示打法详情区，打法卡片直接作为页面入口。

## 10 个打法

1. 引凤计划
2. 钩子品引流法
3. 全生命周期广告法
4. 货品矩阵打法
5. 内容营销打法
6. CPS 打法
7. 站外引流法（CID）
8. 京东指数
9. POP 京仓京配
10. 价促场打法

打法名称后续应集中维护在 JavaScript 数据对象中，便于替换成最终业务文案。

## 交互

- “引凤计划”跳转至 `https://joyspace.jd.com/pages/Fk2oSZ1L7udhX93QEbDf`。
- 其余 9 个打法保留卡片与标题，但暂不配置跳转。
- 页面不依赖接口，入口配置使用本地 JavaScript 数据。

## 响应式布局

### 桌面端

- 页面内容居中，并设置合理最大宽度。
- 主视觉保持原始比例完整展示。
- 11 个打法尽量单行排列。
- 打法卡片采用 5 列网格。

### 移动端

- 主视觉保持比例，不裁切核心内容。
- 11 个打法框横向滚动，不压缩到无法阅读。
- 打法卡片改为 2 列网格，极窄屏幕改为单列。
- 点击区域满足触屏操作需求。

## 技术方案

- 使用 Vite 提供 npm 开发与构建命令。
- 使用原生 HTML、CSS、JavaScript。
- 最终部署目标为京 ME 客户端内的 H5 网页应用。
- Vite 构建使用相对资源路径，避免应用发布目录变化导致静态资源失效。
- 页面不依赖客户端登录、分享或通讯录能力，因此当前版本不引入京 ME JSAPI SDK。
- 适配移动端 WebView 安全区。
- 项目结构保持简单：
  - `index.html`
  - `src/style.css`
  - `src/main.js`
  - `public/assets/hero.png`
- `npm run dev` 启动本地开发环境。
- `npm run build` 生成可部署的静态文件。

## 数据与状态

- JavaScript 中维护一个包含 10 项的打法数组。
- 每项包含打法名称和可选跳转地址字段。
- 页面启动时渲染全部卡片，无运行时选中状态。

## 异常处理

- 主视觉图片加载失败时显示替代文字和红色背景。
- 案例数据缺失字段时显示空字符串，不输出 `undefined`。
- 索引越界时回退到打法 1。

## 验收标准

- 页面可以通过 npm 安装、启动和构建。
- 10 个打法框均展示完整。
- 仅引凤计划具备跳转能力，其他 9 个处于待补充状态。
- 桌面和手机宽度下无横向页面溢出；仅打法导航自身允许横向滚动。
- 主视觉使用用户提供的 `104.PNG`。
- 页面没有分类筛选区域和右侧信息框。
