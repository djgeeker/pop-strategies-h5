# 京东家群 POP 商家 11 大打法案例库 H5 设计规格

## 目标

开发一个可在 npm 环境运行、可部署到 H5 平台的单页专题页面。页面使用原生 HTML、CSS 和 JavaScript，不依赖前端框架。

## 视觉依据

- 主视觉：`/Users/jiahaodong/Downloads/104.PNG`
- 内容结构参考：`/Users/jiahaodong/Downloads/105.PNG`
- 页面整体采用京东红、白、浅灰配色，延续主视觉的促销专题风格。

## 页面结构

1. 顶部为全宽主视觉图。
2. 主视觉下方为 11 个打法选项框，不设置右侧信息框。
3. 不设置业务分类、筛选标签、搜索或其他入口。
4. 11 个打法选项下方展示当前打法唯一对应的案例。

## 11 个打法

1. SEO 优化
2. CPS 推广
3. 流量获取
4. 促销活动
5. 内容营销
6. 转化提升
7. 站内外投放
8. 排名提升
9. 广告投放
10. 直播带货
11. 社群营销

打法名称后续应集中维护在 JavaScript 数据对象中，便于替换成最终业务文案。

## 交互

- 默认选中打法 1。
- 点击任一打法框后：
  - 更新选中状态。
  - 在当前页面原位切换到该打法的案例。
  - 不跳转到新页面。
- 每个打法仅展示一个案例。
- 案例内容区包含案例标题、简述、背景、执行动作、核心亮点和结果数据。
- 页面不依赖接口，案例内容使用本地 JavaScript 数据渲染。

## 响应式布局

### 桌面端

- 页面内容居中，并设置合理最大宽度。
- 主视觉保持原始比例完整展示。
- 11 个打法尽量单行排列。
- 案例区采用左侧视觉封面、右侧文字详情的双栏布局。

### 移动端

- 主视觉保持比例，不裁切核心内容。
- 11 个打法框横向滚动，不压缩到无法阅读。
- 案例区改为单列，上方为案例视觉区，下方为文字详情。
- 点击区域满足触屏操作需求。

## 技术方案

- 使用 Vite 提供 npm 开发与构建命令。
- 使用原生 HTML、CSS、JavaScript。
- 项目结构保持简单：
  - `index.html`
  - `src/style.css`
  - `src/main.js`
  - `public/assets/hero.png`
- `npm run dev` 启动本地开发环境。
- `npm run build` 生成可部署的静态文件。

## 数据与状态

- JavaScript 中维护一个包含 11 项的案例数组。
- 每项包含打法名称、案例标题、描述、背景、动作、亮点和结果字段。
- 页面仅维护一个当前选中索引。
- 切换时重新渲染选项状态与案例内容。

## 异常处理

- 主视觉图片加载失败时显示替代文字和红色背景。
- 案例数据缺失字段时显示空字符串，不输出 `undefined`。
- 索引越界时回退到打法 1。

## 验收标准

- 页面可以通过 npm 安装、启动和构建。
- 11 个打法框均可点击。
- 每个打法均切换到不同的单个案例内容。
- 桌面和手机宽度下无横向页面溢出；仅打法导航自身允许横向滚动。
- 主视觉使用用户提供的 `104.PNG`。
- 页面没有分类筛选区域和右侧信息框。
